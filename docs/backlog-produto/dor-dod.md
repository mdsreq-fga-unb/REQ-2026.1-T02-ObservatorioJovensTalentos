# Definition of Ready (DoR) e Definition of Done (DoD)

!!! info "Unidade 2"
    Esta seção será preenchida na **Unidade 2**.

## Definition of Ready (DoR)

O **DoR** define os critérios que uma história de usuário deve atender **antes de ser incluída em uma iteração** de desenvolvimento. Uma história está "pronta para desenvolvimento" quando:

- [ ] A história está escrita no formato padrão (Como / Quero / Para que)
- [ ] Os critérios de aceitação estão definidos e são verificáveis
- [ ] A história foi estimada pela equipe
- [ ] Não há dependências bloqueantes não resolvidas

## Definition of Done (DoD)

O **DoD** define os critérios que uma história de usuário deve atender **para ser considerada concluída**. Uma história está "pronta para entrega" quando:

- [ ] O código foi implementado e revisado por pelo menos um membro da equipe
- [ ] Os testes unitários foram escritos e estão passando
- [ ] A funcionalidade foi validada com o cliente ou representante
- [ ] A documentação foi atualizada
- [ ] O código foi integrado à branch principal sem conflitos
