# Requisitos Não Funcionais

!!! info "Unidade 2"
    Esta seção será preenchida na **Unidade 2**.

Os requisitos não funcionais descrevem **como o sistema deve se comportar** — restrições de qualidade, desempenho, segurança e usabilidade da plataforma.

Serão documentados utilizando o modelo **URPS+** (Usabilidade, Confiabilidade, Performance, Suportabilidade e categorias adicionais).
