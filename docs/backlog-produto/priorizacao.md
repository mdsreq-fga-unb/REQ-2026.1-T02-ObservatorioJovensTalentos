# Priorização do Backlog

!!! info "Unidade 2"
    Esta seção será preenchida na **Unidade 2**.

A priorização do backlog será realizada utilizando o modelo **MoSCoW**, que classifica os requisitos em quatro categorias:

| Categoria | Descrição |
|-----------|-----------|
| **Must Have** | Requisitos essenciais para o MVP — sem eles o produto não funciona |
| **Should Have** | Requisitos importantes, mas que podem ser adiados sem comprometer o MVP |
| **Could Have** | Requisitos desejáveis, implementados apenas se houver tempo disponível |
| **Won't Have** | Requisitos fora do escopo desta versão, mas que podem ser considerados no futuro |

A priorização será realizada em conjunto com o cliente durante os workshops de priorização previstos na Fase de Inception do processo DAD/Kanban.
